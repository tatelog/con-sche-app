# Con-Scheとは

**Con-Sche（コンスケ）** は、建設業向けのネットワーク工程表（アロー・ダイヤグラム / ADM）作成アプリです。**無料で利用でき、工程表データはお使いの端末にのみ保存されます**（サーバーには送信されません）。

## 主な機能

### 工程表作成・編集

- **ADMネットワーク工程表** — アロー・ダイヤグラム方式で工程表を作成
- **クリティカルパス自動計算（CPM）** — ES/EF/LS/LF/TF/FFを自動算出、クリティカルパスを赤色表示
- **歩掛による工期計算** — 数量・人数・歩掛から作業日数を自動計算
- **カレンダー管理** — 祝日・会社休日・イベント（検査・搬入・式典等）を設定、工期計算に自動反映
- **進捗線（雷線）** — 工程の進捗状況を視覚的に表示
- **パス選択モード** — 経路全体を選択してフロートの範囲内で一括移動
- **テキストボックス** — 横書き・縦書きのテキストを自由配置

### 分析・可視化

- **進捗曲線（Sカーブ）** — 累計進捗率を時系列で表示、計画と実績を比較

### 保存・エクスポート

- **ローカル自動保存** — ブラウザ内（IndexedDB）に自動保存
- **Con-Scheファイル** — フルパッケージ（.csa）/ ライト（.csl）形式で書き出し・読み込み
- **PDF出力** — A1〜A4サイズ、表題欄・押印欄カスタマイズ対応
- **CSV出力** — アクティビティ一覧・CPM結果を含むデータ出力

### 連携API

外部システムやAIエージェントから、CPM計算・工程ファイル変換を呼び出せます。利用者登録時に発行されるAPIコードで、月間無料枠の範囲ですぐに試せます。詳細は[APIドキュメント](https://con-sche.tatelog.biz/api-docs)を参照してください。

:::info 準備中の機能
BIMモデル（IFCファイル）からの数量取り込みは現在準備中です。
:::

## ネットワーク工程表の基本概念

### イベントノード（結合点）

丸印（○）で表される、作業の開始点・終了点です。それぞれに番号が付与され、CPM計算による**最早結合点時刻（ET）**と**最遅結合点時刻（LT）**が表示されます。

### アクティビティ（作業）

矢印（→）で表される個々の作業です。イベントノード間を結び、**作業名**と**所要日数**が表示されます。

### ダミーアクティビティ

作業の実体はなく、依存関係のみを示す点線矢印です。所要日数は0日です。

### クリティカルパス

工程表内の最長経路で、プロジェクト全体の工期を決定します。クリティカルパス上の作業が遅れると、工期全体が遅れます。Con-Scheではクリティカルパスが赤色で強調表示されます。

### バーチャート（ガントチャート）との違い

| 項目 | ネットワーク工程表（ADM） | バーチャート |
|------|--------------------------|-------------|
| 依存関係 | 矢印で明確に表現 | 表現しにくい |
| クリティカルパス | CPMで自動算出 | 判別困難 |
| 工期短縮の検討 | フロート分析で容易 | 困難 |
| 資源配分 | 歩掛計算と連動可能 | 視覚的に把握しやすい |

### CPM（クリティカルパス法）で計算される値

| 略称 | 名称 | 説明 |
|------|------|------|
| ES | 最早開始時刻 | 最も早く作業を開始できる時刻 |
| EF | 最早完了時刻 | 最も早く作業が完了する時刻 |
| LS | 最遅開始時刻 | 工期を遅らせずに開始できる最も遅い時刻 |
| LF | 最遅完了時刻 | 工期を遅らせずに完了できる最も遅い時刻 |
| TF | トータルフロート | 後続作業への影響なく遅らせられる日数 |
| FF | フリーフロート | 直後の作業に影響なく遅らせられる日数 |
